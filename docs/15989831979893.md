# 分片

## 主分片
用于解决数据水平扩展问题。
- 一个分片是一个运行的Lucene的实例。
- 主分片数在索引创建时指定，后序不允许修改，除非Reindex。

## 副本
用于解决数据高可用问题。副本是主分片的拷贝。
- 副本分片数可以动态调整。
- 增加副本数，一定程度上提高服务的可用性「吞吐速度」

## 分片的设定【不太懂】
- 分片数设置过小。
    - 后续无法增加节点实现水平扩展。
    - 单个分片的数据量太大，导致数据重新分配耗时。
- 分片数设置过大。
    - 影响搜索结果的相关性打分，影响统计结果的准确性。
    - 单个节点上过多的分片，会导致资源浪费，同时影响性能。
