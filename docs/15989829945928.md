# 总结

## 代码标准
- 可维护性： bug 容易修复，修改、添加功能能够轻松完成。
- 可读性：代码是否符合编码规范、命名是否达意、注释是否详尽、函数是否长短合适、模块划分是否清晰、是否符合高内聚低耦合。
- 可扩展性：代码预留了一些功能扩展点。
- 灵活性：易扩展、易复用或者易用。
- 简洁性：思从深而行从简。
- 可复用性：解耦、高内聚、模块化、单一职责。
